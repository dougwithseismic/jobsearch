# Web App

Next.js 16 company swipe tool and research dashboard for job search. Keyboard-driven Tinder-style UI for reviewing 147 companies, with per-company detail pages, HN research, website screenshots, and a printable CV page.

Author: Doug Silkstone

## Dev Setup

```bash
# From monorepo root
pnpm dev --filter=web     # http://localhost:3000

# Or directly
pnpm --filter=web build
pnpm --filter=web lint
pnpm --filter=web check-types
```

Requires Node >= 18, pnpm. Puppeteer is used for website screenshots (needs a Chromium install).

## Pages

### `/` ŌĆö Company Swipe Tool

Tinder-style card interface for reviewing companies. Each card shows company name, category, description, role, location, salary, funding, team size, tech stack, reputation grade, and fit score (1-5).

**Keyboard controls:**
- `ŌåÆ` or `Y` ŌĆö Yes
- `ŌåÉ` or `N` ŌĆö No
- `Ōåō` or `M` ŌĆö Maybe
- `Backspace`, `U`, or `Z` ŌĆö Undo last decision

Also has click buttons. Swipe animations on decide. Progress bar at top tracks reviewed count and yes/maybe tallies.

After all companies are reviewed, shows a results view with tabs (Yes / Maybe / No / All) and export to JSON or CSV.

A research panel loads alongside each card:
- Website screenshot (via Puppeteer, cached to `public/screenshots/`)
- Hacker News mentions (via Algolia API, cached to `research-cache.json`)
- Quick links to Google News, LinkedIn, Glassdoor, GitHub searches

### `/company/[slug]` ŌĆö Company Detail Page

Server-rendered page for each company. Shows full metadata grid, fit score, decision badge, and the company dossier (markdown rendered with `react-markdown` + `remark-gfm`).

Dossiers are loaded from `journey/dossiers/{slug}.md`. If none exists, shows a prompt to run `/company-deep-dive`.

Slugs are generated by `lib/slugify.ts` ŌĆö lowercase, `&` to `and`, `.` to `-`, strip non-alphanumeric.

### `/cv` ŌĆö CV Page

Client-side CV viewer with variant selector (outreach, rankacy, default). Loads CV data from `/api/cv?variant=`. Print-friendly ŌĆö designed for "Save as PDF" with no margins and background graphics enabled.

## API Routes

### `GET/POST/DELETE /api/decisions`

Read, merge, or clear the decisions file. Persists to `decisions.json` at the monorepo root.

- `GET` ŌĆö returns full `Record<companyId, "yes" | "no" | "maybe">`
- `POST` ŌĆö merges body into existing decisions
- `DELETE` ŌĆö resets to `{}`

### `GET /api/research?company=`

Hacker News search proxy. Queries `hn.algolia.com` for top 5 stories matching company name. Results cached in `research-cache.json` at monorepo root.

Returns: `{ title, url, points, numComments, createdAt }[]`

### `GET /api/screenshot?url=`

Puppeteer-based website screenshot service. Captures 1280x800 viewport, saves PNG to `public/screenshots/{hostname}.png`. Serves cached version on subsequent requests.

Returns: `{ src: "/screenshots/hostname.png" }`

### `GET /api/cv?variant=`

Serves CV JSON from `assets/cv-{variant}.json`. Falls back to `assets/cv.json` if variant file doesn't exist. Variants: `outreach`, `rankacy`, `default`.

## Data Model

### `app/data/companies.ts`

TypeScript array of 147 companies. Each entry:

```ts
interface Company {
  id: number;           // 1-147 (gap at 136-138)
  name: string;
  category: string;     // "Music Tech", "Games", "Creative Tools", etc.
  description: string;
  location: string;
  workModel: string;    // "Remote", "Hybrid", "Remote Options"
  role: string;
  salaryRange: string;
  website: string;
  funding: string;
  teamSize: string;
  techStack: string;    // comma-separated
  whyCool: string;
  reputation: string;   // e.g. "A ŌĆö Strong Glassdoor reviews"
  fitScore: number;     // 1-5
  notes: string;
}
```

Categories include Music Tech, Games, Creative Tools, Travel, Developer Tools, and others. Companies are sorted by category.

## File-Based Persistence

No database. Everything reads/writes flat files:

| File | Location | Purpose |
|------|----------|---------|
| `decisions.json` | monorepo root | Swipe decisions (`{ "1": "yes", "2": "no", ... }`) |
| `research-cache.json` | monorepo root | HN search result cache |
| `public/screenshots/` | `apps/web/` | Cached website PNGs |
| `journey/dossiers/{slug}.md` | monorepo root | Company research dossiers |
| `assets/cv-*.json` | monorepo root | CV data variants |
